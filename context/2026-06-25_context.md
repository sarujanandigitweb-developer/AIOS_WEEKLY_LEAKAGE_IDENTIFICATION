## PostgreSQL Structure Check
The 26-column activity-memory schema already exposes free-text linkage columns (`source_skill_file`, `evidence_refs`, `files_touched`). A day's activity rows already carry `activity_date` + `deliverable_id`, which key 1:1 to a day context file. → **NO NEW COLUMN REQUIRED.**

## Linkage Recommendation
- **Preferred:** add the per-day context path to **`evidence_refs`** (append `; context/<date>_context.md`) for each day's activity rows — non-destructive, queryable, keeps the SKILL file as the primary `source_skill_file`.
- **Alternative:** set `source_skill_file = context/<date>_context.md` where no SKILL file applies. Avoid this when a SKILL file already occupies that column.
- **Do NOT** create a `context` table or a context-tracking column — the file path lives in existing text columns; the file on disk is the single source.

## Duplicate-Truth Assessment
🟢 **GREEN** — context files are a **day-level narrative summary** of activity rows that already exist; storing only a *file path reference* (not the narrative) in `evidence_refs`/`source_skill_file` adds a pointer, not a second copy of the data. No business/leakage data is duplicated; no new storage object is created.
